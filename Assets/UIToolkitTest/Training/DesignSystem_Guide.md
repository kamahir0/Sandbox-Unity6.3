# デザインシステム設計指針

このプロジェクトでは、UI Toolkit の保守性と一貫性を高めるため、接頭辞を用いた **4層構造のデザインシステム** を採用しています。

## 接頭辞の役割

| 接頭辞 | 分類 | 役割 | 例 |
| :--- | :--- | :--- | :--- |
| **`t-`** | **Theme** | 配色、フォント、視覚効果。全体のスタイルを決定。 | `.t-root`, `.t-surface` |
| **`l-`** | **Layout** | 構造。位置、パディング、境界線など「箱」の定義。 | `.l-window`, `.l-screen` |
| **`c-`** | **Component** | 個別のパーツ。Button, TextField などの具体的な見た目。 | `.c-button`, `.c-input` |
| **`u-`** | **Utility** | 汎用的な単機能。中央寄せ、非表示、細かなマージン調整。 | `.u-center-content` |

## 文字色・背景色指定漏れを防ぐ仕組み (`.t-surface`)

`Sample2` のように「構造（ルート）」と「実体（ウィンドウ）」が分離している場合、ルートに背景色を指定すると重なって不都合が生じたり、指定が漏れたりします。
これを解決するため、**「面（Surface）」** という概念を導入しています。

1.  **`.t-root` (継承の源泉)**:
    全ての UXML のルートに付与。文字色の継承のみを保証し、背景は持ちません。
2.  **`.t-surface` (実体のある面)**:
    背景色を持つべき要素（全画面背景、ウィンドウパネルなど）に付与。

### 使い分けの例
- **全画面 UI**: ルートに `.t-root` と `.t-surface` を両方付与。
- **ウィンドウ UI**: ルートに `.t-root` を付与し、中央のパネルに `.t-surface` を付与。

この使い分けをルール化することで、どのような構造のUIでも「どこが背景を持つべきか」が明確になり、デザインミスを防げます。

## コンポーネントのカスタマイズ

ボタンの色などは、ベースの `.c-button` クラスに加えて、`.c-button--primary` のような接尾辞（Modifier）を組み合わせて表現します。
```xml
<ui:Button class="c-button c-button--primary" />
```
