# Unity Source Generator Reader

This skill enables you to find, list, and read C# source code files automatically generated by Unity's Source Generators (e.g., MessagePack, VContainer, localization tools). Because these files are not stored in the `Assets/` directory but rather deep in OS-specific temporary folders, you must follow specific steps to locate them.

## RULES — Always Follow These

1. **Identify the OS first**: Source generated files are stored in completely different locations depending on the operating system. Always verify the current OS environment (Windows, macOS, or Linux) before attempting to search.
2. **List before reading**: Source generators can create hundreds of files. **Never** attempt to `cat` or read all files at once. Always list the files first, verify their paths and sizes, and then read only the specific files the user needs or that are relevant to the current debugging context.
3. **Respect file limits**: When listing files, limit your output to a reasonable number (e.g., max 50 files) to avoid flooding the context window.
4. **Compile first**: If the directories do not exist or are empty, remind the user to open the Unity Editor and trigger a compilation (or run `unicli exec Compile --json` if `unicli` is available).

## Target Directories by OS

Depending on the OS, target the following root directories for your search:

* **Windows**:
  * `%LOCALAPPDATA%\Temp\SourceGeneratedDocuments`
  * Typically translates to `C:\Users\<Username>\AppData\Local\Temp\SourceGeneratedDocuments`
* **macOS**:
  * `$TMPDIR/SourceGeneratedDocuments` (Usually under `/var/folders/.../T/`)
  * Fallback: `~/Library/Caches/SourceGeneratedDocuments`
* **Linux**:
  * `~/.cache/SourceGeneratedDocuments`
  * Fallback: `/tmp/SourceGeneratedDocuments`

## Key Workflows

### 1. Locating the Directory and Listing Files

Use standard shell commands to find the directory and list `.cs` files. Apply filtering by filename (pattern) or generator name (which is usually the first sub-directory name inside the hash folder).

**macOS / Linux (Bash):**

```bash
# Find the base directory
SG_DIR="${TMPDIR:-/tmp/}SourceGeneratedDocuments"
if [ ! -d "$SG_DIR" ]; then SG_DIR="$HOME/Library/Caches/SourceGeneratedDocuments"; fi

# List up to 50 .cs files, optionally filtering by generator name (e.g., "MessagePack")
find "$SG_DIR" -type f -name "*.cs" | grep -i "MessagePack" | head -n 50

```

**Windows (PowerShell):**

```powershell
# Find the base directory
$sgDir = Join-Path $env:LOCALAPPDATA "Temp\SourceGeneratedDocuments"

# List up to 50 .cs files, optionally filtering by generator name
if (Test-Path $sgDir) {
    Get-ChildItem -Path $sgDir -Recurse -Filter "*.cs" | 
    Where-Object { $_.FullName -match "MessagePack" } | 
    Select-Object -First 50 FullName, Length, LastWriteTime
} else {
    Write-Output "Directory not found. Please compile the Unity project."
}

```

### 2. Reading a Specific Generated File

Once you have identified the exact path of the target `.cs` file from the list step, read its contents to analyze the generated code.

**macOS / Linux (Bash):**

```bash
# Replace with the actual file path found in step 1
cat "/var/folders/xx/yyyy/T/SourceGeneratedDocuments/ProjectHash/MessagePack.SourceGenerator/GeneratedResolver.cs"

```

**Windows (PowerShell):**

```powershell
# Replace with the actual file path found in step 1
Get-Content "C:\Users\Username\AppData\Local\Temp\SourceGeneratedDocuments\ProjectHash\MessagePack.SourceGenerator\GeneratedResolver.cs" -Raw

```

## Tips

* The directory structure usually follows this pattern: `.../SourceGeneratedDocuments/<ProjectHash>/<GeneratorName>/<FileName>.cs`.
* If a user asks "Show me the generated code for X", first find the files matching "X", present the list, and then read the most relevant file.
* Generated code is read-only. Do not attempt to modify these files; they will be overwritten on the next Unity compilation. To change the generated code, you must modify the source files (attributes, partial classes) in the `Assets/` directory that trigger the generation.
