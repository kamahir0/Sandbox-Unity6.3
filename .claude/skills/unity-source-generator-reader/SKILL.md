# Unity Source Generator Reader

Generated `.cs` files live in OS-specific temp directories, not in `Assets/`.

## Directory Locations

| OS      | Path |
|---------|------|
| Windows | `%LOCALAPPDATA%\Temp\SourceGeneratedDocuments` |
| macOS   | `$TMPDIR/SourceGeneratedDocuments` |
| Linux   | `~/.cache/SourceGeneratedDocuments` |

Structure: `.../SourceGeneratedDocuments/<ProjectHash>/<GeneratorName>/<File>.cs`

## Workflow

### 1. Detect OS & locate directory

```bash
# macOS / Linux
SG_DIR="${TMPDIR:-/tmp}/SourceGeneratedDocuments"
[ ! -d "$SG_DIR" ] && SG_DIR="$HOME/Library/Caches/SourceGeneratedDocuments"
echo "$SG_DIR"
```

```powershell
# Windows
$sgDir = Join-Path $env:LOCALAPPDATA "Temp\SourceGeneratedDocuments"
Test-Path $sgDir
```

### 2. List files first (always, max 50)

```bash
find "$SG_DIR" -type f -name "*.cs" | grep -i "<GeneratorName>" | head -n 50
```

```powershell
Get-ChildItem -Path $sgDir -Recurse -Filter "*.cs" |
  Where-Object { $_.FullName -match "<GeneratorName>" } |
  Select-Object -First 50 FullName, Length
```

### 3. Read only the target file

```bash
cat "<exact-path-from-step-2>"
```

## Notes

- **Directory empty?** → Ask user to compile in Unity Editor (or run `unicli exec Compile --json`).
- Files are **read-only**. To change generated output, edit the source in `Assets/` (attributes, partial classes), not the generated files.
